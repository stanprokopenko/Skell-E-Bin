Overview of the Vulnerability

There's a new bug affecting Apple's M1, M2, and M3 chipsets, which is a flaw in the silicon design of the CPU, making it unpatchable without physically replacing the CPU. This bug requires local access to exploit, meaning an attacker needs to have access to the computer to utilize it. The bug is part of research from various universities and researchers, compiled into a white paper, proof of concept, and vulnerability collectively referred to as "Go Fetch." The vulnerability allows an unprivileged process to read data from another process, including cryptographic key information, by exploiting side-channel cache-based memory attacks.

Technical Background

Side-channel attacks exploit the implementation details of a system to leak information, using indirect measures like time, heat, or sound. Cache memory, which is faster than RAM and sits between the CPU and RAM, is shared among processes and has been an attack surface for hackers through side-channel attacks. Constant time programming is a CPU architecture design that aims to make operations take the same amount of time regardless of cache hits or misses to prevent timing-based side-channel attacks.

The "Go Fetch" Bug

The "Go Fetch" bug exploits data memory dependent prefetchers (DMPs) in Apple silicon, which prefetch memory addresses without validating if they are actual pointers, not adhering to constant time programming. The vulnerability allows a process to monitor another process's memory access patterns through cache timing, potentially leaking cryptographic keys used by various libraries. The demonstration of the attack shows an attacker process filling up the cache and observing memory eviction to reveal information about the CPU and the cryptographic operations of another process.

Complexity of Exploitation

The complexity of the attack involves understanding CPU architecture and cryptographic implementation, highlighting the depth of knowledge required to discover and exploit such vulnerabilities.
